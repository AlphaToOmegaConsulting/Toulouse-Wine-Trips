# .codex

This folder stores project-specific Codex setup and working trackers for this repository.

## What lives here

- `config.toml`: project-level Codex settings
- `tracking/`: live working sheets for client onboarding and follow-up work
- `templates/`: reusable blank checklists for future clients

## Why this exists

OpenAI Codex documentation supports project-scoped settings in `.codex/config.toml`.
The main long-term repo instructions still live in `AGENTS.md`.

For this project, `.codex` is used as the clean working area for operational tracking.
